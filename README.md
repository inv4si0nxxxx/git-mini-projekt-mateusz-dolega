## Cel projektu
Projekt powstał po to, żeby zrobić prostą stronę i przy okazji ogarnąć i nauczyć się podstaw gita i githuba.

## Opis projektu
Mini projekt jest prostą stroną/notesem o nauce Gita. Zawiera pliki HTML, CSS, JS, dokumentację w katalogu `docs`.

## Jak uruchomić projekt
1. Pobierz repozytorium z GitHub.
2. Otwórz plik `index.html` w przeglądarce.
3. Sprawdź, czy strona, style i skrypt działają poprawnie.

## Struktura plików
* `index.html` - strona startowa mini projektu.
* `style.css` - wygląd strony.
* `script.js` - prosta interakcja i licznik zadań.
* `data/zadania.json` - przykładowe dane z zadaniami.
* `docs/` - krótka dokumentacja projektu.
* `pliki-do-branchy/` - materiały pomocnicze do pracy na trzech branchach.

## Wymagane branche
* `main`
* `feature/opis-readme`
* `feature/komendy-git`
* `feature/interfejs-zadania`

## Moje najważniejsze komendy
Użyte w projekcie kluczowe komendy Git to:
* `git init` – inicjalizacja nowego repozytorium.
* `git status` – podgląd stanu zmodyfikowanych plików.
* `git add` – dodawanie konkretnych plików do poczekalni.
* `git commit` – zapisanie zmian w historii z opisem.
* `git switch` – przełączanie się pomiędzy gałęziami roboczymi.
* `git merge` – scalanie kodu z gałęzi bocznych do głównej main.
* `git push` – wysyłanie lokalnych zmian na serwer GitHub.
